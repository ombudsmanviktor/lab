--- title: FAQ description: Perguntas Frequentes sobre o SucupiraLAB e o pqLAB ---

O que são o SucupiraLAB e o pqLAB?

O SucupiraLAB é uma aplicação web de gestão acadêmica voltada para pesquisadores, docentes e coordenadores de pós-graduação, projetada para organizar e acompanhar atividades de rotina de gestão de projetos de pesquisa exigidas pela Plataforma Sucupira da CAPES. O pqLAB, por sua vez, é uma ferramenta web de organização de pesquisa qualitativa que centraliza em uma única interface seis instrumentos de trabalho acadêmico: diário de campo, listas, tarefas, favoritos, fichamentos e planos de curso, todos conectados por links internos navegáveis. Ambos os softwares foram desenvolvidos por Viktor Chagas e pelo coLAB/UFF com auxílio do Claude Code Sonnet 4.6.


Como os dados dos usuários são armazenados?

O SucupiraLAB não utiliza banco de dados próprio: todos os dados são armazenados diretamente em um repositório GitHub privado do próprio usuário, garantindo total controle, privacidade e portabilidade das informações. O pqLAB adota a mesma lógica: os dados são armazenados diretamente no repositório GitHub privado do usuário, sem servidor intermediário.


Quais são os principais módulos de cada ferramenta?

O SucupiraLAB oferece módulos para prestações de contas, discursos qualificados, projetos financiados, orientações em andamento, nucleação, produção científica, internacionalização e fluxo de submissões de manuscritos. O pqLAB, voltado à pesquisa qualitativa, organiza-se em torno de diário de campo, listas e memorandos, tarefas, favoritos, fichamentos e planos de trabalho ou de disciplinas, além de uma visualização em mapa mental gerada automaticamente a partir dos links internos criados pelo usuário.


Como instalar e começar a usar os softwares?

A instalação de ambos os softwares segue o mesmo fluxo e dispensa terminal ou build local. O usuário deve fazer um fork do repositório no GitHub, ativar o GitHub Pages nas configurações do fork, criar um repositório privado para armazenar os dados e gerar um Personal Access Token com escopo repo. Em seguida, basta acessar a URL publicada pelo GitHub Pages e preencher as credenciais na tela de login.
